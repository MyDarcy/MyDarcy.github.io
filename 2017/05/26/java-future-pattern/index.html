Future模式浅析

发表于 2017-05-26 | 分类于 Java | 阅读次数:
字数统计: 2.4k 字 | 阅读时长 ≈ 11 分钟

Future模式的核心是异步调用。有时候调用要很久才能返回，但是我们并不急于想要结果。因此我们可以让被调用者立即返回，让它在后台慢慢处理这个请求。对于调用者而言，则可以先处理一些其他任务，在真正需要数据的场合再去尝试获得需要的数据。

对于Future模式而言，虽然它无法立即给出想要的结果，但是，他会立即返回给你一个契约，将来，凭着这个契约可以取重新获取你想要的信息。

Future模式

异步调用时序图

Future模式的主要角色

  • Main：系统启动，调用Client发出请求
  • Client: 返回Data对象，立即返回FutureData对象，并开启ClientThread装配ReadData
  • Data: 返回的数据接口
  • FutureData: Future数据，构造很快，但只是一个虚拟数据，需要装配ReadData
  • ReadData: 真实数据，构造比较慢。

Future模式的简单实现

  • 核心接口Data，代表客户端希望获取的数据。Future模式中，这个Data接口有两个重要的实现。分别是ReadData– 代表真实的数据，即最终要获得的，有价值的信息。另一个就是FutureData– 是可以立即返回的，是用来提取RealData的一个契约。
  • FutureData实现了一个快速返回的RealData包装，作为RealData的虚拟实现。当使用FutureData的getResult()方法时候，如果实际的数据没有准备好，那么程序就会阻塞，等待RealData准备好并且注入到FutureData中，才会最终返回数据。

JDK中的Future模式

  • Future接口类似契约，通过它，可以得到真实的数据。RunnableFuture继承了Future和Runnable两个接口，其中run()方法用于构造真实的数据。它有一个具体的实现FutureTask类。FutureTask中有一个内部类Sync，一些具体的工作会委托Sync来完成。而Sync类最终会调用Callable接口，完成实际数据的组装工作。 (JDK1.8中并没有发现)
  • Callable接口只有一个方法call(), 会返回实际要构造的数据，这个Callable接口也是这个Future框架和应用程序之间的重要接口。如果要实现自己的业务系统。通常需要构造自己的Callable对象。此外，FutureTask类也和应用密切相关，通常我们会使用Callable实例构造一个FutureTask实例。并且把它提交给线程池。
