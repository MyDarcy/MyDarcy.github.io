本文是Java并发基础的第一篇，主要包括线程状态以及线程的基本操作，包括新建、终止、中断、wait/notify、join/yield等。

Java并发基础-part-1

进程

  • 进程（Process）是计算机程序关于某个数据集合上的一次运行活动，是系统进行资源调度的基本单位；进程可以理解为线程的容器；程序是指令、数据以及其组织形式的描述，而进程是程序的实体；
  • 线程是轻量级的进程，是程序执行的最小单位；线程间的切换和调度的开销远远小于进程；

线程状态

  • NEW ： 线程刚刚创建，还没有开始执行；
  • RUNNABLE： 线程的start()方法调用，线程处于执行态；
  • BLOCK：线程执行过程中遇到了synchronized同步块，线程暂停执行，直到获得锁；
  • WAITING：线程进入一个无限期的等待；(通过wait()方法等待的线程在等待notify()方法，通过join()方法等待的线程则会等待目标线程执行结束；一旦等待到目标事件，线程会再次执行，进入RUNNABLE状态)
  • TIMED_WAITING：线程进入一个有时间限制的等待；
  • TERMINATED：线程执行结束；

线程的基本操作

新建线程

  • start方法： 会新建一个线程，并且让这个线程执行run方法；
  • run方法: 没有创建新的线程，而是在当前线程中调用run()方法, 只是作为一个普通方法被调用；
  • 所以不要使用run方法来启动新的线程, 它只会在当前线程中直接调用run()方法中的代码；
  • Thread(Runnable target): 传入Runnable接口的实例，在start方法调用时候，新的线程就会执行Runnable.run()方法；
  • 默认的Thread.run()就是直接调用内部的Runnable接口，所以使用Runnable接口告诉线程应该做什么才是更合理的做法；

终止线程

  • Thread中的stop()方法废弃不用，因为会强行把执行到一半的线程终止，可能导致数据不一致的问题；Thread.stop()方法在结束线程时，会直接终止线程，并且会立即释放这个线程持有的锁，而这些锁恰恰是用来维持对象的一致性的。如果这个时候写线程写入数据写到一半，被强行终止，那么这个对象就会处于不一致的状态，而且此时锁对象已经释放了，另外一个等待该锁的读线程就会读到这个不一致的对象；
  • Thread.yield(): 就是说当一个线程使用了这个方法之后，它就会把自己CPU执行的时间让掉，让自己或者其它的线程运行。

A hint to the scheduler that the current thread is willing to yield its current use of a processor. The scheduler is free to ignore this hint.

线程中断

  • 上一小节描述了stop方法的缺陷，以及使用了自定义的方式来完善线程的退出机制；但其实JDK其实是通过更强大的线程中断来支持线程退出；
  • 线程中断并不会使得线程立即退出，而只是给线程发一个通知(有人希望你退出)；但是目标线程接收到通知以后如何处理，则完全是目标线程自行决定的。
  • Thread.sleep()方法由于中断而抛出异常，此时，它会清除中断标记，如果不佳处理，那么下一次循环开始的时候，就无法捕获这个中断，因此在异常处理中再次设置中断标记位；

等待(Wait)和通知(notify)

  • 在一个对象A上调用wait()方法，当前线程就会在该对象上等待，直到其他线程调用了对象A的notify()方法。
  • wait()和notify()方法的工作原理: 如果一个线程调用了object.wait()，那么它就会进入object对象等待队列– 因为可能有多个线程在同时等待某个对象；当object.notify()方法被调用时，它就会从这个等待队列中随机选择一个线程并将其唤醒(并不是FIFO的服务方式)；
  • Object.wait()方法并不是可以随时调用的，必须包含在对应的synchronized语句中; 无论是wait()还是notify()还是notifyAll()还是他们的重载版本，在调用之前必须先获得对象的锁;
  • Obejct.wait()和Thread.sleep()方法都会让线程等待一段时间，但是wait()方法会释放目标对象的锁，但是Thread.sleep()方法不会释放任何资源；

挂起(suspend)和继续执行线程(resume)

  • 声明deprecated， 不推荐使用；因为suspend()方法在导致线程暂停的同时，并不会取释放任何锁资源，而其他任何线程想要使用被它占用的锁，都会被阻塞无法正常执行；直到对应的线程调用上调用了resume()方法，被挂起的线程才能继续执行；但是如果resume方法意外的在suspend方法之前就执行了，那么被挂起的线程很难有机会再次执行；而且它占用的锁不会被释放，而且被挂起的线程的状态仍然是Runnable的；

等待线程结束(join)和让步(yield)

  • 多线程中，一个线程的输入可能非常依赖另一个或者多个线程的输出，那么这个线程就需要等待依赖线程执行完毕，才能继续往下执行；join()方法实现这样的功能；
  • join()方法的本质是让调用线程在当前线程对象上进行等待。当线程执行完毕之后，被等待的线程会在退出之前调用notifyAll方法通知所有的等待线程继续执行；(不要在应用程序中，在Thread对象实例上调用wait或者notify等方法；因为可能影响系统API的工作或者被系统API所影响。)
  • 在线程A中调用了线程B的Join()方法，是A线程进入wait状态，直到线程B执行完毕后，才会继续执行线程A。
  • Join方法实现是通过wait（Object提供的方法）。当main线程调用t.join时候，main线程会获得线程对象t的锁（wait 意味着拿到该对象的锁),调用该对象的wait(等待时间)，直到该对象唤醒main线程 ，比如退出后。这就意味着main 线程调用t.join时，必须能够拿到线程t对象的锁。(有点不是很好理解)
